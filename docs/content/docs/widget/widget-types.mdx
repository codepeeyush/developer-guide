---
title: Widget Types
description: The widget ships three distinct UI layouts ‚Äî Tabs, Compact, and Search ‚Äî sharing the same socket layer and core logic but differing in structure and navigation.
---

The active layout is determined by `useChatbot().layout.type`, which comes from the API settings response. The `Widget` component renders the matching layout:

```typescript
// src/widget/index.tsx
switch (layout.type) {
  case LayoutTypeE.tabs:    return <TabsWidget />;
  case LayoutTypeE.compact: return <CompactWidget />;
  case LayoutTypeE.search:  return <SearchWidget />;
}
```

<Callout type="info">
Override the layout locally during development with `VITE_WIDGET_LAYOUT_TYPE=compact|tab|search` in your `.env.local`. See [Build & Local Dev](./build) for all env vars.
</Callout>

## Tabs Widget

**Location:** `src/widget/views/tabsWidget/`

The most feature-rich layout. Presents a multi-tab interface inside a chat window.

### Tabs
| Tab | Path | Description |
|---|---|---|
| Home | `/home` | Welcome screen, quick actions |
| Messages | `/messages` | Chat conversation list + active chat |
| Profile | `/profile` | Visitor info, language picker |
| Help | `/help` | Help articles, search |

### Navigation
**File:** `src/widget/views/tabsWidget/store/navigationStore.ts`

Uses a lightweight internal router:
```typescript
navigationStore.navigate({ route: "/messages", params: { sessionUid } })
navigationStore.goBack()
```

### Stores
- `tabsStore` ‚Äî list of open tabs, active tab index
- `chatStore` ‚Äî messages per `sessionUid` (supports multiple sessions simultaneously)
- `navigationStore` ‚Äî current route + params stack

### Structure
```

‚îú‚îÄ‚îÄ Wrapper.tsx         ‚Üê socket listener setup
‚îú‚îÄ‚îÄ layout/             ‚Üê tab bar, shell chrome
‚îî‚îÄ‚îÄ tabs/
    ‚îú‚îÄ‚îÄ home/
```

**When to use:** Chatbots that need more than just a conversation ‚Äî helpdesk articles, user profiles, multiple sessions. See [Helpdesk](./helpdesk) for the Help tab details.

## Compact Widget

**Location:** `src/widget/views/compactWidget/`

A minimal, single-conversation chat window. The most common layout.

### Structure
```
compactWidget/
‚îú‚îÄ‚îÄ Wrapper.tsx           ‚Üê socket listeners, main orchestrator
‚îú‚îÄ‚îÄ store/
‚îÇ   ‚îú‚îÄ‚îÄ compactStore.ts   ‚Üê messages + session state
‚îÇ   ‚îî‚îÄ‚îÄ socket.ts         ‚Üê direct socket ‚Üí store handlers
‚îî‚îÄ‚îÄ (components)/
    ‚îî‚îÄ‚îÄ Chatbot/
        ‚îî‚îÄ‚îÄ index.tsx     ‚Üê chat UI, input, message list
```

### Stores
- `compactStore` ‚Äî single flat `messages[]` array + `sessionDetail`

### IFrame Support
The compact widget can render inside an `<iframe>` for stricter host page isolation. Configured via layout settings ‚Äî see [Embedding](./embedding).

**When to use:** Default for most chatbot use cases. Simple, focused, minimal overhead.

## Search Widget

**Location:** `src/widget/views/searchWidget/`

A full-width, search-first experience. Appears fixed at the bottom-center of the page (not in a corner). Designed for knowledge-base or documentation search flows.

### Structure
```
searchWidget/
‚îú‚îÄ‚îÄ Wrapper.tsx
‚îú‚îÄ‚îÄ store/
‚îÇ   ‚îú‚îÄ‚îÄ searchStore.ts
‚îÇ   ‚îî‚îÄ‚îÄ socket.ts
‚îî‚îÄ‚îÄ (components)/
    ‚îú‚îÄ‚îÄ SearchWidget/     ‚Üê entry/landing view
    ‚îî‚îÄ‚îÄ SearchChat.tsx    ‚Üê active conversation view
```

### Stores
- `searchStore` ‚Äî identical structure to `compactStore`

Uses a CSS variable bridge to adapt the host page's styling into the widget. See [Theming](./theming) for details.

**When to use:** Search-heavy products where the primary interaction is "find an answer" rather than conversational chat.

## What's Shared Across All Three

| Concern | Shared? | Details |
|---|---|---|
| Socket connection | Yes | Same `socketManager` singleton |
| `useMessageReceived` hook | Yes | Same streaming/non-streaming logic |
| `streamingStore` | Yes | Same streaming state management |
| `aiActionsStore` | Yes | Same AI action handling |
| `ChatItem2` component | Yes | Same message renderer |
| `MarkdownMessage` | Yes | Same markdown/code rendering |
| Settings / `useChatbotStore` | Yes | Same global config |
| Message stores | **No** | Each widget has its own isolated store |
| Navigation | **No** | Tabs has router; Compact/Search do not |
| Socket listener setup | **No** | Each `Wrapper.tsx` registers its own |
